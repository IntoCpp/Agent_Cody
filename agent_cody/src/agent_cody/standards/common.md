# General Coding Standards

**All code you produce must follow these clean‑code rules:**

1. **Small, focused functions** — each does one thing well.  
2. **Readable names** for all variables, functions, classes, and constants. No abbreviations.  
3. **Code must read like text** — clear flow, no clever tricks, no magic numbers.  
4. **Every function has a comment** describing what it does, its inputs, and its outputs.  
5. **Every loop has a comment** explaining its purpose.  
6. **Always initialize variables** before use.  
7. **Prefer clarity over complexity** — early returns, minimal nesting.  
8. **Language‑agnostic** — adapt these rules to any programming language.  
9. **Output full, clean, maintainable code** with all required comments.
